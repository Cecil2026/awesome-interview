# 通用面试清单

无论你跟的是哪份路线图，都可以用这份清单。它从首场 onsite 前两周开始，结束于面试后的复盘。打印它、复制到 Notion，或者 fork 仓库后在 GitHub 上勾选——只要能让你为自己负责即可。

节奏要点：递减，而非递增。大多数候选人在最后几天疯狂突击，结果筋疲力尽地走进面试场。拿到 offer 的候选人则是逐步收缩。

## 面试前 2 周

这是新内容还能起作用的最后窗口。把它用在流程安排和最后的模拟上，而不是新材料上。

- [ ] 与招聘联系人书面确认面试形式：轮次数量、每轮时长、所用编码工具（CoderPad、HackerRank、CodeSignal、共享文档、白板）、允许使用的语言、每轮考察什么
- [ ] 确认面试官的姓名和职位 — 在 LinkedIn 上调研每一位，以便提出有针对性的问题
- [ ] 安排 2-3 次模拟面试（同伴、付费服务如 interviewing.io 或 Pramp、或一位你信赖的资深工程师）
- [ ] 从你的备战中找出最薄弱的两个话题领域，仅在本周安排额外的专项训练
- [ ] 起草你最强的 5 个 STAR 故事，确保每个至少映射到以下任意 2 项：领导力、冲突、模糊性、失败、技术深度、跨团队协作、对客户的影响
- [ ] 写下 5-7 个要问面试官的问题，按面试官类型分组（同级工程师、招聘经理、跨级、招聘联系人）。问题要针对该团队，不能泛泛。
- [ ] 确认流程：面试所在时区、日历邀请已接受、视频会议工具已安装并测试
- [ ] 如果是 onsite：确认地址、停车或公共交通、着装要求、前台接待你的人、所需的身份证件
- [ ] 如果是远程：在实际平台（Zoom、Meet、Teams）上与朋友测试摄像头、麦克风和屏幕共享

## 面试前 1 周

仅做轻量复习。不学新材料。好好睡觉。

- [ ] 重读自己路线图里的笔记 — 不要开任何新的章节或课程
- [ ] 每天从你最难的类别中重做 1 题。最多 25 分钟。结束就停。
- [ ] 每天大声把 STAR 故事各读一遍
- [ ] 如果还能约到，再做最后 1 次模拟 — 完整流程或单轮，由你选
- [ ] 选好你的面试服装（哪怕是远程 — 穿戴得体会改变你的姿态和语气）。熨平。挂好。
- [ ] 再次确认你的视频环境正常：脸上有光、麦克风清晰、背景整洁、第二屏幕位置合适以便眼睛靠近摄像头
- [ ] 如果使用独立白板或草稿工具（Excalidraw、OneNote、iPad），打开并确认它按你预期的方式工作
- [ ] 规划面试当天的饮食 — 易消化、不尝新、不油腻
- [ ] 如果备战期间咖啡因摄入过量，减回正常基线
- [ ] 本周开始每晚睡 8 小时。前一晚才开始已经太晚。

## 面试前一天

午饭后停止学习。你的大脑在夜里巩固记忆，而不是在晚上 11 点。

- [ ] 仅在上午对 STAR 故事做轻量复习（最多 30 分钟）
- [ ] 重读职位描述与团队介绍，让背景清晰
- [ ] 为每类面试官重读 3 个准备好的问题
- [ ] 准备好衣服
- [ ] 准备好面试当天的随身包：水瓶、轻便零食（香蕉、燕麦棒 — 不要会掉渣或弄脏的）、笔记本、两支可用的笔、充电器、远程的话还需要耳机
- [ ] 如果是远程：确认日历邀请里的链接正确、链接能正常打开、并能提前 5 分钟加入
- [ ] 如果是 onsite：在包里装好身份证、水、零食、笔记本、笔、充电器以及任何需要的资料
- [ ] 晚上 6 点后不做技术学习。做一些真正放松的事 — 散步、和喜欢的人共进晚餐、看场电影。不要看烧脑的电影。
- [ ] 不喝酒
- [ ] 手机关机，放到另一个房间，10 点前熄灯。目标 8 小时睡眠。

## 面试当天

计划：平静到场，准时离开，不出意外。

**早晨**
- [ ] 在第一场面试至少 2 小时前起床
- [ ] 正常早餐。吃点含蛋白的。多喝水。
- [ ] 简单拉伸或短途散步。如果适合你，可静坐 20 分钟 — 任何能让你回到身体感觉的方式都行。
- [ ] 开始前做 3 分钟的盒式呼吸（吸气 4 秒、屏 4 秒、呼气 4 秒、屏 4 秒）

**面试前（30 分钟）**
- [ ] 上洗手间
- [ ] 水瓶装满放在手边
- [ ] 零食放在手边
- [ ] 笔记本翻开，笔准备好
- [ ] STAR 故事摘要卡放在视线余光处（每个故事一行 — 情境、任务、行动、结果）
- [ ] 当前面试官要问的问题列表可见
- [ ] 如果是远程：提前 5 分钟进入会议，摄像头与麦克风打开，面试官进入时微笑
- [ ] 如果是 onsite：在前台报到时间前 10 分钟到达大楼

**每场面试中**
- [ ] 开场确认面试官的姓名和职位
- [ ] 编码轮：写代码前先把问题口头澄清清楚。复述输入、输出和约束。询问边界情况。
- [ ] 系统设计：先澄清范围 — 功能需求、非功能需求、规模估算。不要直接跳到架构。
- [ ] 行为面试：选一个故事，用 90-120 秒按 STAR 讲完，然后停下来等追问。克制继续说下去的冲动。
- [ ] 边想边说。沉默在面试官听来等于"卡住"，哪怕你其实在推进。
- [ ] 不知道就说不知道，然后给出你会如何查清楚的方案。永远不要硬装。
- [ ] 末尾留 3-5 分钟问问题。提前准备好。

**面试之间**
- [ ] 每次休息都去洗手间
- [ ] 每次休息都喝水
- [ ] 时间段较长就吃一口零食
- [ ] 不要在脑子里回放上一轮。它已经结束。下一轮才是你能影响的。
- [ ] 轮次之间做 3 次盒式呼吸

**整场流程结束后**
- [ ] 如果可以，去外面走一走
- [ ] 吃一顿正餐
- [ ] 24 小时内：给招聘联系人发一封简短的感谢邮件，并（如果符合该公司文化）给每位面试官也发一封。两句话。这封邮件里不要追问后续问题。

## 如果面试表现糟糕

有些面试就是会很糟。这是预期之内，并非灾难。

- [ ] 无论如何也要做复盘。尤其是这种。让你痛过的数据最有价值。
- [ ] 识别这是知识缺口、沟通缺口、紧张缺口还是契合度缺口 — 不同问题有不同解药
- [ ] 不要陷入恶性循环。一次糟糕的流程只是某一瞬间的信号，不代表你这个人。
- [ ] 在一周内安排下一次模拟。向前推进是最好的解毒剂。
- [ ] 如果未来一周内还有别的面试，不要过度纠正。相信你的备战。出场就行。

## 如果面试表现良好

- [ ] 同样要做复盘。胜利与失败一样有教益，但前提是写下来。
- [ ] 精确记下什么管用 — 用词、框架、故事选择。重复使用。
- [ ] 不要松懈。offer 签字之前都不算到手。
- [ ] 如果拿到 offer，不要立刻回答"是"。至少留 24 小时。谈判。

## 最后提醒

面试是一种技能。你可以是工作能手却不擅长面试，也可以善于面试却没准备好做这份工作。这份清单调的是前一个变量。你的路线图和你的职业生涯调的是后一个。

祝你好运。
