# 为 awesome-interview 贡献内容

感谢你帮助让这份面试备战资源变得更好。无论是纠错、新增题目、改进答案、新增模拟面试记录，还是补充翻译，都非常欢迎。

本仓库是纯 Markdown 加上少量仅依赖标准库的 Python 脚本。无需安装任何东西 —— 只要你的 PATH 上有 Python 3，就能运行一切。

## 贡献方式

- **修正或改进答案** —— 让表述更清晰、更准确，补一个权衡点，或换一个更好的例子。
- **新增题目** —— 加到某个 `knowledge/` 主题题库、公司文件或行为面试集合中。
- **新增模拟面试** —— 一份带复盘的完整对话记录（见下文）。
- **翻译** —— 补全或改进对应的 `*.zh.md` 镜像。
- **工具** —— 改进 `tools/` 里的脚本（保持仅依赖标准库）。

## 仓库约定

### 条目结构

每个问答条目都使用相同的标题格式，这样随机选题器、每日一题工作流和校验器才能统一地解析任意文件：

```markdown
### N. 题目标题

**Answer:** ……正文……

**Key points:**
- 要点
- 要点
```

- 编号 `N` 是一个指引，而非契约 —— 可以自由重新编号，但同一文件内的条目必须**连续**（1、2、3……）且不重复。
- `knowledge/`、`behavioral/` 和 `mock-interviews/` 使用上面的散文式结构。
- **公司文件**（`interviews/companies/*.md`）额外使用结构化元数据行，包括
  `**Tags:** #algorithm | #coding | #system-design`。算法/编码类条目应包含复杂度
  （一行 `**Complexity:**` 或答案中内联的 `O(...)`）。校验器对公司文件强制要求这一点。
- 算法题需附带 **Python、TypeScript 和 Java** 三种实现。阅读器会把连续的多语言代码块渲染成
  可切换的标签页，因此请在每个代码块前单独一行标注 `**Python:**`、`**TypeScript:**`、`**Java:**`。

### 双语镜像

本地化内容存放在并行的 `*.zh.md` 文件中，按条目编号与英文文件对齐。如果你给
`knowledge/backend.md` 加了 `### 7.`，就要给 `knowledge/backend.zh.md` 加上对应的
`### 7.`。校验器会在数量不匹配时发出警告。

### 模拟面试

模拟对话记录适合从头到尾阅读。每一份都包含：

- 一段 **Setup**（背景：场景 + 谁在面试谁）。
- 完整的 **Transcript**，使用加粗的 `**INTERVIEWER:**` / `**CANDIDATE:**` 标签，
  并用斜体表示舞台提示。
- 复盘部分：**What went well**、**What could've been stronger**、**Key takeaways**。
- 系统设计类轮次还要附一张 **Reference architecture** ASCII 架构图。

保持每轮简短（通常 1-4 句），对任何真实人物做匿名处理，并至少安排一处候选人被质疑、
需要随之调整的时刻。

## 提交 PR 之前

1. 新增或修改你的 `### N. Question` 条目（如适用，同时更新其 `*.zh.md` 镜像）。
2. 刷新搜索索引：
   ```bash
   python tools/build_index.py
   ```
3. 运行校验器并修复它报告的所有问题：
   ```bash
   python tools/validate.py          # 报告问题
   python tools/validate.py --strict # 将警告当作错误处理
   ```
4. 同时提交你的 Markdown 改动**和**刷新后的 `docs/questions.json`。
   如果你忘了刷新索引，CI 会拦下这个 PR。

校验器会检查编号、双语对齐、公司文件的标签/复杂度、内部链接以及索引是否最新。
CI 会在每次 push 和 PR 时自动运行它。

## 风格建议

- 要具体。一个犀利的权衡或一个小而完整的示例，胜过含糊其辞。
- 诚实地说明某个方案在什么情况下是错的、或不太合适 —— 这些细微之处正是价值所在。
- 不要专有或机密材料。只收录公开已知的面试题。
- `*.md` 用美式英语，`*.zh.md` 用简体中文。

## 行为准则

请保持尊重与建设性。默认善意，把反馈聚焦在内容上，并乐于帮助新人。
任何形式的骚扰或歧视都不受欢迎。
