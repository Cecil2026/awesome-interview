# AI 与机器学习面试题

100 道高频面试题，覆盖机器学习基础、深度学习、NLP、LLM、生成式 AI、MLOps、模型服务、评估与 AI 系统工程。

---

### 1. 偏差-方差权衡

**答案：** 泛化误差可分解为偏差（模型假设错误导致的欠拟合）、方差（对训练集波动的敏感性，即过拟合）以及不可约噪声。高偏差模型（如对非线性数据使用线性回归）会系统性地遗漏信号；高方差模型（如小数据上的深层决策树）会记忆噪声。通过控制模型容量、正则化（L1/L2、dropout、early stopping）、增加数据（降低方差）或集成（bagging 降方差，boosting 降偏差）来管理这一权衡。诊断方法：比较训练误差与验证误差——差距大表示方差问题；两者都高表示偏差问题。现代过参数化网络通过 double descent 复杂化了经典 U 型曲线，但该框架仍指导日常模型选择和调试。

**要点：**
- 训练/验证差距用于诊断偏差与方差。
- 正则化与更多数据降低方差。
- 增加容量或特征降低偏差。
- 过参数化场景下出现 double descent。

---

### 2. 过拟合及其防止方法

**答案：** 过拟合是模型捕获了训练噪声而非潜在模式，导致训练误差低但测试误差高。原因：相对于数据容量过大、特征过多、训练过久、验证集泄漏，或类别不平衡让模型走捷径。缓解方法：更多或更优质的数据、数据增强、更简单的模型、L1/L2 权重衰减、dropout、early stopping、交叉验证以获得诚实估计、batch normalization、label smoothing 以及集成。在深度学习中，数据增强和 dropout 是首选；经典机器学习中，正则化和特征选择占主导。最终评估前务必保留一个未被触碰的测试集，并将训练-验证曲线发散视为过拟合的标准信号。

**要点：**
- 训练低、验证高 = 过拟合。
- 正则化、dropout、early stopping 是核心工具。
- 更多/更干净的数据胜过精巧技巧。
- 永远不要在测试集上调参。

---

### 3. 训练/验证/测试集划分与交叉验证

**答案：** 将数据划分为训练集（拟合参数）、验证集（调超参数和选模型）和测试集（最终一次无偏估计）。常见比例：大数据 70/15/15 或 80/10/10；数据稀缺时使用 k 折交叉验证（k=5 或 10），让每个样本既参与训练又参与验证。分类任务按标签分层采样以保持类别比例。时间序列必须使用前向链式（滚动窗口）划分，绝不能随机——过去必须预测未来。分组数据（同一用户多行）按组划分以避免泄漏。反复用测试集做决策会悄悄泄漏信息；如需迭代，应另保留一个新的测试集。

**要点：**
- 测试集只在最后用一次。
- 分类要分层，时间序列要按时序。
- 分组划分防止用户级泄漏。
- 小数据用 k 折，大数据用 holdout。

---

### 4. 交叉验证策略

**答案：** k 折将数据切成 k 等份，每次用 k-1 折训练、1 折验证，跨折平均指标。分层 k 折保持每折的类别分布，是分类任务的默认选择。Leave-one-out（LOO）即 k=n；方差大且耗时，但对小数据无偏。重复 k 折跨多次随机划分平均以稳定估计。嵌套 CV 外层用于评估，内层用于超参搜索，避免两者结合时的乐观偏差。时间序列用 TimeSeriesSplit（扩展或滑动窗口）。Group k-fold 尊重分组边界（如每患者）。根据数据依赖结构和要回答的问题来选择方案。

**要点：**
- 分层 k 折是分类的安全默认。
- 嵌套 CV 用于诚实的超参选择。
- 时间序列需要前向链式划分。
- 分组折避免聚类数据中的泄漏。

---

### 5. 正则化：L1 vs L2 vs 弹性网

**答案：** L2（岭回归）在损失中加 lambda * sum(w^2)，将权重平滑收缩至零，处理多重共线性良好，且有稳定的闭式解。L1（lasso）加 lambda * sum(|w|)，将部分权重精确推至零，实现隐式特征选择并产生稀疏模型。弹性网用混合参数 alpha 结合两者，兼具 L1 的稀疏性、L2 的稳定性，并可选择相关特征组（L1 单独使用会任意挑一个）。L2 适合稠密信号和稳定预测，L1 适合需要稀疏性或可解释性时，弹性网适合特征相关时。神经网络中 L2（权重衰减）是标配；AdamW 将其与自适应梯度解耦以获得更清洁的行为。

**要点：**
- L1 = 稀疏；L2 = 收缩；弹性网 = 两者兼具。
- L1 在相关特征间任意选择。
- AdamW 将权重衰减与自适应步长解耦。
- Lambda 通过交叉验证调优。

---

### 6. 特征工程与特征选择

**答案：** 特征工程从原始数据创造预测信号：类别编码（one-hot、target、嵌入）、数值缩放、分箱、交互、多项式展开、对数/Box-Cox 变换、日期分解和领域聚合。特征选择剪除冗余或噪声输入以减少过拟合并提速，方法包括 filter（相关性、互信息、卡方）、wrapper（前向/后向选择、RFE）和 embedded（L1 正则化、树特征重要性）。现代深度学习通过表示学习将大量工程交给模型，但表格问题仍取决于特征。注意泄漏：任何使用目标统计量的变换（target encoding、均值填充）必须仅在训练折上拟合。

**要点：**
- 表格机器学习的成败在特征。
- 泄漏是隐形杀手；变换只在训练集上拟合。
- 嵌入替代高基数类别的 one-hot。
- L1 与树重要性是实用的 embedded 选择器。

---

### 7. 类别不平衡的处理

**答案：** 当一类占绝大多数（如欺诈占 0.1%）时，朴素模型预测多数类即可获高准确率却无实用召回。补救：重采样（SMOTE/ADASYN 过采样少数类、欠采样多数类或两者结合）、损失中加类别权重（sklearn class_weight='balanced'）、focal loss 聚焦难例、用精确率-召回率曲线调阈值而非默认 0.5，并选择反映成本的指标（PR-AUC、F1、recall@k、成本敏感指标）。分层划分保持比例一致。注意：SMOTE 在高维空间生成不真实的合成点；划分前过采样会泄漏。极端不平衡下，异常检测建模或两阶段模型（高召回过滤器 + 精确分类器）常优于纯重平衡。

**要点：**
- 不平衡下准确率会撒谎；用 PR-AUC、F1、recall@k。
- 类别权重比合成过采样更省更稳。
- 调阈值常是最简单的大收益。
- SMOTE 应在 CV 折内做，不能在划分前。

### 8. 损失函数：何时用哪种

**答案：** 回归：MSE（L2）重罚离群值，零附近梯度快；MAE（L1）对离群值鲁棒；Huber 介于两者之间。二分类：二元交叉熵（log loss）输出概率，hinge loss 用于 SVM 式最大间隔。多分类：搭配 softmax 的分类交叉熵。有序回归：累积链接或自定义有序损失。不平衡：focal loss 降低易例权重。排序：pairwise（RankNet）或 listwise（LambdaRank、ListNet）。嵌入：对比、triplet、InfoNCE。检测：focal + IoU/GIoU 等组合。选择应匹配输出类型与错误成本结构；分类不要用 MSE，回归不要用交叉熵。校准也依赖损失——log loss 天然产生良好校准的概率。

**要点：**
- 损失须匹配输出类型与错误成本。
- 交叉熵校准概率，hinge 不会。
- Huber/MAE 适用于离群多的回归。
- focal loss 适用于极端类别不平衡。

### 9. 分类指标：精确率、召回率、F1、ROC-AUC、PR-AUC

**答案：** 精确率 = TP/(TP+FP)（预测为正中有多少正确）；召回率 = TP/(TP+FN)（实际正例中抓住多少）。F1 是两者调和平均，适合两者并重时。ROC-AUC 通过 TPR vs FPR 度量所有阈值下的排序质量；对类别平衡不敏感，但在严重不平衡数据上可能虚高。PR-AUC 绘制精确率 vs 召回率，正例稀少时更具信息量。准确率在不平衡下会误导。按成本选指标：医疗筛查重召回；垃圾邮件过滤重精确；排序系统用 AUC 或 NDCG。务必报告多个指标和混淆矩阵，并按运营约束调阈值。

**要点：**
- 精确率与召回率通过阈值权衡。
- 严重不平衡下 PR-AUC > ROC-AUC。
- 仅看准确率几乎从不够。
- 混淆矩阵显示真实错误构成。

---

### 10. 概率校准

**答案：** 当预测概率 p 表示事件确实以约 p 的频率发生时，模型即为校准的。即使用交叉熵训练的树集成、SVM 和神经网络也可能未校准（深度网络常过自信，提升树常不自信）。用可靠性图与期望校准误差（ECE）诊断。小数据或二分类用 Platt scaling（在 logit 上做逻辑回归）修正；数据充足时用 isotonic 回归做灵活的非参数校准。Temperature scaling——将 logit 除以一个学习的标量 T——是深度网络的标准修正，保持准确率且极廉价。务必在留出集上校准（非训练集），并在数据分布偏移时重新校准。

**要点：**
- ECE 与可靠性图量化校准误差。
- Temperature scaling：单参数，对网络大有裨益。
- Isotonic 灵活但比 Platt 需要更多数据。
- 模型选择后，在留出集上校准。

### 11. ROC 与 PR 曲线：何时用哪个

**答案：** ROC 随阈值变化绘制真阳性率 vs 假阳性率；AUC 概括排序质量并对类别平衡不变。PR 绘制精确率 vs 召回率，对正例比例敏感，正例稀少时（欺诈、疾病、点击预测）更具信息量。严重不平衡下，ROC-AUC 可能看上去很好（如 0.95），但实用召回处的精确率惨不忍睹；PR-AUC 会暴露此问题。平衡问题或需要跨数据集做平衡不变比较时用 ROC；不平衡和检索类问题用 PR。两者都跨阈值汇总——仍需按业务成本选择运营阈值。

**要点：**
- ROC-AUC 与平衡无关；PR-AUC 与之有关。
- 正例稀少时用 PR-AUC。
- 两者汇总阈值选择；仍须挑一个。
- 与部署阈值下的混淆矩阵配套使用。

### 12. 生成式 vs 判别式模型

**答案：** 判别模型直接学 P(y|x)：逻辑回归、SVM、随机森林、多数神经网络。它们把容量集中在决策边界，通常用更少数据获得更好分类准确率。生成模型学 P(x,y) 或 P(x|y) 并通过贝叶斯定理分类：朴素贝叶斯、高斯混合、HMM、VAE、扩散。它们能生成新样本、处理缺失数据、检测异常（低 P(x)）。仅需标签时判别式是默认选择；需要数据合成、半监督学习或对输入有原则性不确定性时生成式获胜。现代 LLM 技术上是生成式（学 P(token | 上下文)），但既用于生成也用于分类。

**要点：**
- 判别：P(y|x)，分类准确率更好。
- 生成：P(x,y)，能采样并检测异常。
- LLM 是生成式，但常被当作判别式使用。
- 生成式在半监督和小标签场景闪光。

---

### 13. 参数化 vs 非参数化模型

**答案：** 参数化模型假设固定函数形式和固定参数数（线性/逻辑回归、朴素贝叶斯、固定架构的神经网络）。训练拟合参数；预测快且省内存。非参数化模型容量随数据增长：KNN、决策树、核 SVM、高斯过程。它们假设更少，数据充足时可拟合任意函数，代价是预测较慢、扩展性更差。"非参数化"是误名——它们也有参数，只是数量无界。先验强或需快速推理时用参数化；关系未知且数据中等时用非参数化。树集成结合非参数化灵活性与合理扩展性，主宰表格机器学习。

**要点：**
- 参数化 = 容量固定；非参数化 = 随数据增长。
- 树与 KNN 是非参数化主力。
- 非参数化推理通常较慢。
- 树集成常是表格默认。

### 14. 机器学习项目生命周期

**答案：** 问题界定（模型改进什么决策、指标是什么、基线是什么）→ 数据采集与标注 → 探索性数据分析 → 特征工程与划分 → 基线模型 → 用恰当验证迭代建模 → 错误分析 → 校准与阈值调整 → 留出集离线评估 → 与当前生产模型的线上 A/B → 监控（数据漂移、性能、公平性）→ 重训练节奏 → 最终下线。多数失败发生在上游：错误的问题、泄漏的数据、离线/线上指标不匹配、无监控。把多数时间花在数据和评估上，而非追求 0.5% AUC 提升。模型是最便宜的部分。

**要点：**
- 问题界定与指标设计决定成败。
- 线上 > 线下；信任模型前先做 A/B。
- 监控与重训练是生命周期的一部分。
- 数据质量胜过花哨算法。

### 15. 数据泄漏

**答案：** 泄漏发生在训练集外的信息悄悄进入，产生乐观验证而到生产中崩塌。形式：目标泄漏（特征源自或代理了标签，如用 "was_refunded" 预测 "fraud"）、训练-测试污染（划分前在全集上预处理）、时间泄漏（用未来信息预测过去）、组泄漏（同一用户出现在训练与测试中）、跨划分的重复行。症状：验证好得不真实，特征重要性被某可疑列主导。预防：先划分再拟合变换，仅在训练折上拟合，时间相关按时间验证、分组数据按组验证，并批判性地阅读 top 特征。每当离线远胜线上时怀疑泄漏。

**要点：**
- 先划分，后变换。
- 可疑的 top 特征往往 = 泄漏。
- 时间/组感知划分防止隐性泄漏。
- 离线-线上差距是经典症状。

### 16. 逻辑回归

**答案：** 线性模型将 w·x+b 经 sigmoid（二分类）或 softmax（多分类）输出概率，通过最大化对数似然（即最小化交叉熵）拟合。凸优化得唯一解；系数有直接的对数几率解释。用 L1（稀疏）、L2（收缩）或弹性网正则化。在高维稀疏特征（文本、广告）上表现极佳。局限：特征空间中决策边界线性（可用交互或核函数缓解），假设观测独立，对离群值与多重共线性敏感。仍是强基线和生产主力，用于点击预测、信用评分以及任何可解释性和良好校准概率比原始准确率更重要的场景。

**要点：**
- 凸、校准好、可解释。
- 高维稀疏问题的强基线。
- 加交互或核函数突破线性上限。
- 正则化处理共线与过拟合。

---

### 17. 决策树

**答案：** 通过贪心划分递归分割特征空间，分类时最大化信息增益（熵减）或基尼不纯度下降，回归时最小化方差。处理混合类型、缺失值和非线性而无需缩放。浅树易解释。主要弱点：高方差——小数据变化可能产生很不同的树——以及过拟合倾向。用 max_depth、min_samples_leaf、ccp_alpha 剪枝控制。单独的树很少出彩，但它是最强表格方法的基石：随机森林（bagging）和梯度提升树（XGBoost/LightGBM/CatBoost）。贪心划分可能错过特征交互；集成可修复。

**要点：**
- 基于不纯度的贪心划分；无需缩放。
- 单独高方差；需集成才强。
- max_depth 与 min_samples_leaf 控制过拟合。
- 是 RF 与梯度提升的基础。

---

### 18. 随机森林

**答案：** 在 bootstrap 样本上训练去相关的决策树集成（bagging），每次划分仅考虑随机特征子集（分类常用 sqrt(p)，回归 p/3）。预测取平均（回归）或多数投票（分类）。随机特征带来的去相关使集成方差远低于单树。对离群值鲁棒，能处理非线性交互，自带特征重要性和袋外（out-of-bag）误差估计，无需单独验证集。权衡：可解释性不如单树，比线性模型慢，在表格基准上可能不及提升树。是想要强大、低维护、少调优表格模型时的优秀默认。

**要点：**
- Bagging + 随机特征子集 = 去相关树。
- 袋外样本免费给验证估计。
- 比 boosting 调优少；通常稍弱。
- 表格问题的鲁棒默认。

---

### 19. 梯度提升：XGBoost、LightGBM、CatBoost

**答案：** Boosting 顺序拟合树，每棵通过损失的梯度下降校正之前集成的残差。XGBoost 加入二阶梯度、正则化、稀疏感知划分和并行；LightGBM 用基于直方图的划分和按叶生长，对大数据更快；CatBoost 用有序目标编码原生处理类别，避免泄漏。它们主宰表格 Kaggle 比赛与生产表格机器学习。关键超参：树数（在验证集上 early stopping）、学习率（更小 + 更多树 = 更好）、max_depth 或 num_leaves、subsample/colsample、L1/L2 正则。比 RF 训练慢、可解释性差，但通常是表格最强表现者。

**要点：**
- 在残差梯度上顺序拟合树。
- LightGBM 最快；CatBoost 对类别最佳。
- 树数与 LR 共同调优并用 early stopping。
- 表格机器学习的金标准。

---

### 20. 支持向量机

**答案：** 找到最大化类间间隔的超平面；只有支持向量（最近点）决定边界。软间隔参数 C 在间隔宽度与误分类之间权衡。核技巧（RBF、多项式、线性）通过核函数隐式将输入映射到更高维空间，无需显式特征展开即可实现非线性边界。在高维以及特征多于样本时（如 TF-IDF 文本分类）效果好。局限：训练复杂度 O(n^2) 到 O(n^3)，超过约 10 万样本后扩展性差；核选择关键；默认不输出概率（需 Platt scaling）；C 与 gamma 调优重要。如今大多被树集成和神经网络取代，但在小型高维数据集上仍然强势。

**要点：**
- 最大化间隔，由支持向量决定。
- 核技巧实现非线性边界。
- 不可扩展超过约 10 万样本。
- 需 Platt scaling 输出概率。

---

### 21. K 最近邻

**答案：** 惰性学习器：存储所有训练数据；预测时找 k 个最近的训练样本（用欧氏、曼哈顿、余弦等距离），然后多数投票（分类）或平均（回归）。无训练成本，但推理为 O(n)，除非有索引（低维用 KD-tree、ball tree，高维用 ANN 如 HNSW/FAISS）。对特征缩放敏感（务必归一化）、受维度灾难影响（高维下距离失去意义）、对 k 敏感（k 小则高方差，k 大则高偏差）。适合作为简单基线、通过嵌入 + ANN 搜索构建推荐系统以及做异常检测。直接用于分类多被学习模型取代，但在检索中仍是核心。

**要点：**
- 无训练；无索引时推理慢。
- 必须缩放特征；用 CV 选 k。
- 高维下表现差。
- ANN 库（FAISS、HNSW）使其在检索中可扩展。

---

### 22. 朴素贝叶斯

**答案：** 应用贝叶斯定理，并"朴素地"假设给定类时特征条件独立。变体：高斯（连续特征）、多项式（文本词频）、伯努利（二元）。闭式训练（仅计数和归一化），极快极省内存，在文本分类（垃圾邮件、情感）上效果意外地好——独立假设技术上错误但实际无害。校准差（独立假设被破坏导致概率过自信）。是文本的强基线和需要近零成本模型时的好选择。在重要 NLP 任务上多被逻辑回归和基于 Transformer 的分类器超越，但其简洁性使其仍有用。

**要点：**
- 假设特征独立；很少为真但常奏效。
- 一次扫描完成训练；极快。
- 概率未良好校准。
- 文本与小数据分类的最佳基线。

---

### 23. K-means 聚类

**答案：** 将 n 个点划分为 k 个簇，最小化簇内平方距离之和。Lloyd 算法交替：将点分配给最近质心，重新计算质心。用 k-means++ 初始化以更好收敛。需预先选 k（用肘部法、silhouette 分数或 gap statistic）。假设簇为大小相近的球形，对离群值和缩放敏感。Mini-batch k-means 可扩展到大数据。对非凸或密度变化的簇效果差——改用 DBSCAN、HDBSCAN 或谱聚类。实用于客户分群、向量量化、anchor box 选择、图像压缩。务必归一化特征并尝试多个种子。

**要点：**
- 假设球形等大簇。
- k-means++ 初始化避免坏局部最小。
- 用肘部或 silhouette 选 k。
- DBSCAN/HDBSCAN 用于任意形状。

---

### 24. PCA：主成分分析

**答案：** 线性降维，找到捕获最大方差的正交方向（主成分）。通过中心化数据矩阵的 SVD 或协方差矩阵的特征分解计算。特征值给出每个成分解释的方差。用途：可视化（投影到 2-3 维）、去噪、去相关、下游模型前的特征压缩、距离方法加速。局限：仅线性（非线性用 kernel PCA 或自编码器），成分难解释，对特征缩放敏感（先标准化），若被丢弃成分携带标签信息则可能损害监督性能。非线性结构可视化优先用 t-SNE 或 UMAP；带重建的压缩 PCA 仍是主力。

**要点：**
- 沿正交轴最大化方差。
- 先标准化特征。
- 仅线性；非线性用 kernel PCA 或自编码器。
- 按累计解释方差选成分。

---

### 25. t-SNE vs UMAP

**答案：** 两者都是非线性降维，主要用于高维数据（嵌入、单细胞基因组）的 2D/3D 可视化。t-SNE 通过概率相似性保留局部邻域，产生分离良好的簇但扭曲全局结构；用 Barnes-Hut 时为 O(n log n)，跨运行不稳定（perplexity 敏感）。UMAP 用模糊单纯集理论，更快（近线性），更好保留全局结构，并支持样本外投影（可 fit 后 transform 新点）。UMAP 通常是新工作的默认选择。两者都是可视化工具，不是下游模型的特征——投影中的距离无意义，簇可能是 artifact。

**要点：**
- 仅可视化；不要作为机器学习特征。
- UMAP 更快且更好保留全局结构。
- t-SNE 的 perplexity、UMAP 的 n_neighbors/min_dist 是关键旋钮。
- 多次运行与种子以诚实解读。

---

### 26. 反向传播

**答案：** 通过沿计算图按逆拓扑顺序应用链式法则，高效计算标量损失对所有参数的梯度。前向计算激活与损失；反向把 dL/d(输出) 向后传播，在每个节点累加 dL/dparam。成本约为前向的两倍。现代框架（PyTorch autograd、JAX、TensorFlow）动态构图或一次追踪再复用。关键问题：数值稳定性（用 log-sum-exp）、深层网络中梯度消失/爆炸（用 ReLU、batch/layer norm、残差、谨慎初始化缓解）以及内存成本（用梯度检查点缓解）。反向传播"只是"系统化应用链式法则，但支撑了所有现代深度学习。

**要点：**
- 链式法则的反向自动微分。
- 成本约 2 倍前向。
- 梯度检查点处理内存瓶颈。
- 稳定激活与归一化保持梯度流动。

---

### 27. 梯度下降变种：SGD、动量、Adam、AdamW

**答案：** 朴素 SGD 更新 w := w - lr * grad；mini-batch 加入有助于逃离尖锐最小的噪声。动量累积过去梯度的 EMA（beta ~ 0.9），沿一致方向加速。Nesterov 动量在计算梯度前先看一步。Adagrad 用累积平方梯度的逆根缩放每参数 LR（适合稀疏特征，但 LR 衰减至零）。RMSProp 改用 EMA。Adam = 动量 + RMSProp + 偏差校正；非常流行的默认。AdamW 将权重衰减与自适应更新解耦以获得更好泛化，现已成 Transformer 标配。SGD+动量在视觉中常泛化最佳；Adam/AdamW 主宰语言和大规模训练。

**要点：**
- SGD+动量：很多视觉场景下最佳泛化。
- Adam：收敛快，好默认。
- AdamW 修复 Adam 中权重衰减耦合。
- LR 是最重要的单个超参。

---

### 28. 学习率调度

**答案：** 常数 LR 鲜少最优：过高发散，过低爬行。常见调度：阶梯衰减（每 N epoch 降 10 倍）、指数衰减、余弦退火（在训练中平滑降至零）、带温重启的余弦（SGDR）、平台减小（验证停滞时降）。Transformer 标配为前 1-10% 步线性 warmup 后接到峰值分数的余弦或线性衰减。Warmup 避免自适应优化器尚未建立方差估计时的早期不稳定。One-cycle（Smith）在单 epoch 内先升后降 LR 并常更快训练。用学习率查找器挑峰值 LR。调度选择可让最终准确率波动数个百分点。

**要点：**
- Warmup + 余弦是 Transformer 默认。
- LR 查找器廉价挑出好峰值。
- 平台减小对未知场景鲁棒。
- 调度选择常比优化器选择更重要。

---

### 29. Batch Normalization

**答案：** 沿 batch 维归一化层的前激活（减 batch 均值，除以 batch 标准差），然后应用学习的缩放与平移（gamma、beta）。稳定训练、允许更高学习率、起轻度正则化作用并加速 CNN 收敛。推理时使用训练时统计量的滑动平均。陷阱：依赖 batch size（小 batch 或 batch_size=1 微调差）、训练 vs 评估模式不同（常见 bug）、不适合 RNN/Transformer（改用 layer norm）以及训练-评估分布不匹配。替代：layer norm（每样本）、group norm（通道分组）、instance norm（每通道每样本）。BN 仍是 CNN 主干默认；Transformer 与 RNN 用 layer norm。

**要点：**
- 沿 batch 归一化；学习 gamma/beta。
- 小 batch 不好；改用 GroupNorm/LayerNorm。
- 训练/评估模式不匹配是经典 bug。
- CNN 用 BN；Transformer 用 LN。

---

### 30. Layer Normalization

**答案：** 在单个样本内沿特征维归一化，与 batch size 无关。计算为 (x - mean) / std 加学习的缩放与平移，均值和标准差在该层特征上求。用于 Transformer 和 RNN，因为它适合 batch size 1、序列变长输入以及运行 batch 统计无意义的在线推理。Pre-norm（子层前 LayerNorm）让 Transformer 训练更稳，可扩展到数百层；post-norm（原始 Transformer）常需 warmup 技巧。RMSNorm 去掉均值减法和偏置，更快，实际同样有效——用于 LLaMA 和许多现代 LLM。

**要点：**
- 每样本归一化；与 batch size 无关。
- Pre-norm 训练更深更稳。
- RMSNorm：更简单、更快，用于 LLaMA/Mistral。
- Transformer 与 RNN 的默认。

---

### 31. Dropout

**答案：** 训练中以概率 p（通常 0.1-0.5）随机置零每个单元，存活单元按 1/(1-p) 缩放。作为指数多个子网络的集成以及防止协同适应的正则化。推理时无 dropout——完整网络用原权重。在全连接层最有效；带 BN 的卷积网络中需求少，现代 CNN 常不用。Transformer 中 dropout 应用于注意力概率、残差和 FFN。变体：DropConnect（丢权重而非激活）、spatial dropout（卷积中丢整个特征图）、DropPath/stochastic depth（丢整个残差分支）。现代大模型常用小 p（0-0.1），因为数据充足、过拟合较少。

**要点：**
- 用于正则化的随机集成。
- 推理关闭；权重通过 1/(1-p) 缩放补偿。
- 带 BN 与充足数据时不那么关键。
- DropPath/stochastic depth 在现代架构中常见。

---

### 32. 权重初始化

**答案：** 糟糕初始化导致激活消失/爆炸和死亡神经元。Xavier/Glorot 初始化按 sqrt(2 / (fan_in + fan_out)) 缩放权重，假设线性/tanh；He 初始化对 ReLU 按 sqrt(2 / fan_in)。偏置通常初始为 0；LSTM 遗忘门偏置常初始为 1 以开始"记忆"。Transformer 用缩放初始化（带小常数的 Xavier）加精心安排的 LayerNorm 位置。RNN 中正交初始化有用。现代替代包括 LSUV（数据依赖的重缩放）和 Fixup/T-Fixup（去除 BN/LN 需求）。配合合适的初始化和归一化，极深网络可可靠训练；缺一不可，超过约 10 层就爆炸或消失。

**要点：**
- ReLU 用 He 初始化；tanh/线性用 Xavier。
- 偏置归零（LSTM 遗忘门除外）。
- 初始化 + 归一化共同实现深层训练。
- 框架默认合理初始化；不要盲改。

---

### 33. 梯度消失与梯度爆炸

**答案：** 深层网络中，链式法则相乘的梯度可能缩至零（饱和 sigmoid/tanh、小权重）或爆炸（大权重、循环）。消失阻止早期层学习；爆炸产生 NaN。缓解：ReLU 及变体（正向无饱和）、batch/layer 归一化（保持激活良好缩放）、残差连接（梯度高速路）、合适权重初始化（He/Xavier）、梯度裁剪（RNN/LLM 中将范数限于 1.0）、LSTM/GRU 门（cell 状态中的恒定误差传送带）和通过跳跃连接缩短有效深度。现代 Transformer 训练结合 pre-norm、AdamW、warmup 和裁剪。早期发现爆炸容易（loss NaN）；消失更微妙——监控每层梯度范数。

**要点：**
- 症状：loss NaN，早期层权重冻结。
- ReLU + 归一化 + 残差 + 裁剪 = 标准修复。
- LSTM 门专为消失而设计。
- 监控每层梯度范数。

---

### 34. 残差连接（ResNet）

**答案：** 将块输入加到其输出：y = F(x) + x。让梯度通过恒等路径直接流动，使数百乃至数千层网络可训练。块学习从恒等的"残差"增量而非完整映射，更易优化。Pre-activation 变体（块内 BN-ReLU-Conv）泛化更好。残差在 ResNet（视觉）、Transformer（每个子层都是残差）、Diffusion U-Net 等中实现现代深度。无残差时，极深网络在训练准确率（不仅测试）上退化，因为更深假设类更难优化。是 2010 年代最有影响力的单个架构创新之一。

**要点：**
- y = F(x) + x；梯度高速路。
- 实现数百层无退化。
- Pre-activation ResNet 泛化更好。
- 普遍用于现代深度架构。

---

### 35. 注意力机制

**答案：** 计算值的加权组合，权重来自查询与键之间的可学相似度。缩放点积注意力：softmax(QK^T / sqrt(d_k)) V。让每个位置基于内容聚合任意其他位置的信息，取代固定感受野。自注意力的 Q、K、V 都源自同一输入。多头注意力并行运行 h 个不同投影的头并拼接结果——捕获多样关系。原始用于 NMT 中编解码器对齐；Transformer 用自注意力完全取代循环。成本在序列长度上为 O(n^2 * d)，激发了稀疏/线性近似以支持长上下文。注意力是现代 NLP 和日益增长的视觉的基础。

**要点：**
- Softmax(QK^T / sqrt(d_k)) V。
- 多头捕获多种关系类型。
- O(n^2) 成本驱动长上下文研究。
- 自注意力在 Transformer 中取代循环。

---

### 36. 激活函数

**答案：** 让网络能建模非线性的非线性函数。Sigmoid 和 tanh 饱和导致梯度消失——隐藏层很少再用。ReLU max(0, x) 快、正向不饱和，主宰 CNN；有"死亡 ReLU"问题（单元卡在 0）。Leaky ReLU/PReLU/ELU/SELU 用小负斜率补救。GELU x*Phi(x) 更平滑，是 Transformer（BERT、GPT）标配。Swish/SiLU x*sigmoid(x) 类似 GELU，用于 EfficientNet、LLaMA。Softmax 用于输出概率（和为 1）；sigmoid 用于二元输出。现代 LLM 在 FFN 中偏好 GeGLU/SwiGLU 门控变体。按架构惯例与经验证据选择；鲜少进入超参 top 3。

**要点：**
- 默认 ReLU；Transformer 用 GELU/SwiGLU。
- 死亡 ReLU 用 Leaky/PReLU 缓解。
- 多分类输出用 softmax。
- 门控激活（SwiGLU）在现代 LLM 中常见。

---

### 37. CNN：卷积与池化

**答案：** 卷积在输入上滑动小的可学滤波器，计算点积产生特征图。性质：权重共享（每个空间位置同一滤波器，参数远少于全连接）、平移等变（物体在任何位置都被检测）、随深度增长的局部感受野。步幅、填充、空洞控制输出大小与感受野增长。池化（最大、平均）做空间下采样，提供平移不变并降低计算。现代架构常用带步卷积取代池化。1x1 卷积改变通道而不改空间——常用作瓶颈。是图像、音频和日益增长的时间序列模型的基础。ViT 挑战了卷积的必要性，但混合模型常在效率上胜出。

**要点：**
- 权重共享 + 局部性 = 参数效率。
- 池化提供平移不变性。
- 1x1 卷积用于通道混合/瓶颈。
- ViT 有竞争力但卷积在小数据上仍强。

---

### 38. ResNet、EfficientNet、ViT

**答案：** ResNet 引入残差连接，让 50/101/152 层图像分类器成为事实上的 CNN 主干。EfficientNet 用神经架构搜索（NAS）加复合缩放联合增加深度、宽度和分辨率，在准确率/FLOPs 权衡上优于均匀缩放单一维度；使用倒置残差（MBConv）和 SE 块。ViT（Vision Transformer）把图像切成 patch、视作 token 序列、应用标准 Transformer——需要大规模预训练（JFT-300M）才能超过 CNN，但缩放优秀。ConvNeXt 等混合体把 Transformer 时代的技巧用回 CNN。ResNet 仍是小数据强默认；ViT 与后继者（Swin、DINOv2）在大规模下领先。

**要点：**
- ResNet：残差实现深网。
- EfficientNet：NAS + 复合缩放。
- ViT：patch + Transformer；需规模。
- ConvNeXt：现代化 CNN，与 ViT 竞争。

---

### 39. RNN、LSTM、GRU

**答案：** RNN 用每步更新的隐状态处理序列。朴素 RNN 受梯度消失困扰——无法学长依赖。LSTM 加入输入/遗忘/输出门和最少变换流过时间的 cell 状态（恒定误差传送带），支持长程学习。GRU 把遗忘和输入合并为更新门并绑定 cell+hidden；参数更少，性能常与 LSTM 相当。双向变体并行跑前后向并拼接状态（仅非流式任务）。2017-2018 前在 NLP/语音重度使用；多数任务因顺序瓶颈和有限长程建模而被 Transformer 取代。仍用于流式/低延迟和端上场景。

**要点：**
- LSTM 门修复朴素 RNN 的消失。
- GRU 比 LSTM 简单，性能相近。
- 顺序 = 难以在 GPU 上并行。
- 2018 后多被 Transformer 取代。

---

### 40. 为何 Transformer 取代 RNN

**答案：** RNN 顺序处理 token，阻碍 GPU 并行并造成长程梯度问题（即使 LSTM 缓解）。Transformer 用自注意力加位置编码取代循环：每个位置在一次矩阵乘中关注所有其他位置，跨序列完全并行，任意两 token 间路径长度恒定（梯度流远好）。缩放律更平滑、更可预测。代价：序列长度上 O(n^2) 注意力复杂度和二次内存，激发长上下文研究（FlashAttention、稀疏、线性注意力、SSM）。Transformer 也受益于与硬件（矩阵乘密集）和大规模预训练数据集的幸运契合。结果：2018 起 NLP（BERT/GPT）的主宰，并迅速蔓延到视觉（ViT）、语音、生物学。

**要点：**
- 跨序列并行是杀手特性。
- 路径长度恒定改善梯度流动。
- O(n^2) 成本驱动长上下文创新。
- 预训练 + 缩放律让 Transformer 不可阻挡。

---

### 41. Transformer 架构

**答案：** 来自 "Attention Is All You Need"（2017）的编解码器模型。编码器：相同块的堆叠，每块为多头自注意力 + 位置式 FFN，均包裹于残差 + LayerNorm。解码器：以上加掩码自注意力（防止偷看未来）和编解码器交叉注意力。位置编码（正弦或学习）注入顺序。现代变体：仅编码器（BERT）用于理解，仅解码器（GPT）用于生成，编解码器（T5、BART）用于 seq2seq。后续创新：pre-norm、RMSNorm、旋转位置嵌入（RoPE）、分组查询注意力（GQA/MQA）、SwiGLU FFN、FlashAttention。基本块自 2017 以来保持惊人稳定；多数进展来自规模 + 数据 + 小架构调整。

**要点：**
- 注意力 + FFN + 残差 + LayerNorm = 一个块。
- 按任务选编码器、解码器或两者。
- RoPE、GQA、SwiGLU 是现代标准升级。
- 架构自 2017 稳定；规模是真正主力。

---

### 42. 自注意力 vs 交叉注意力

**答案：** 自注意力：Q、K、V 都来自同一序列；让每个 token 关注同上下文中其他。用于编码器（双向，无掩码）和解码器（因果掩码屏蔽未来 token）。交叉注意力：Q 来自一个序列（如解码器），K 和 V 来自另一个（如编码器输出）。用于编解码器 Transformer（T5、BART、原始 Transformer）和条件化（如文生图扩散：U-Net 关注文本嵌入）。现代仅解码器 LLM 仅用带因果掩码的自注意力；编解码器在某些专门任务（翻译、代码编辑）中回归。交叉注意力是注入外部上下文的标准方式（RAG/工具可能通过拼接隐式使用）。

**要点：**
- 自：Q、K、V 同源。
- 交叉：Q 来自一处，K/V 来自另一处。
- 因果掩码 = 解码器；无掩码 = 编码器。
- 交叉注意力是标准条件化机制。

---

### 43. 多头注意力

**答案：** 将 Q、K、V 投影到 h 个子空间（头），每头并行做缩放点积注意力，拼接，再投影回。让模型同时关注不同表示子空间和位置的信息——头可专精于句法、共指、位置、语义。总参数与等价 d_model 的单头相当，但表达力更高。现代效率技巧：多查询注意力（MQA，所有头共享一个 K/V）以小质量代价削减 KV 缓存内存；分组查询注意力（GQA）把头分组共享 K/V——LLaMA-2/3、Mistral 用的甜蜜点。许多头训练后可剪枝；多数冗余。头数通常 d_model / 64。

**要点：**
- 不同子空间的并行头。
- 头常可解释地专精。
- MQA/GQA 减少长上下文推理的 KV 缓存。
- 剪枝显示许多头冗余。

---

### 44. 位置编码：正弦、学习、RoPE、ALiBi

**答案：** 注意力对置换不变——需位置信息。原始 Transformer 加不同频率的正弦向量；学习嵌入（BERT/GPT-2）效果相当，但不能超出训练长度泛化。旋转位置嵌入（RoPE）按位置相关角度旋转 Q 和 K，乘性编码相对位置；泛化更好，是 LLaMA、Mistral、Qwen 的标配。ALiBi 在注意力分上加位置相关线性偏置，不需嵌入，外推良好。NoPE 在某些仅解码器设置中完全去位置编码（因果掩码提供顺序）。长上下文扩展时，带缩放/插值（NTK-aware、YaRN）的 RoPE 让预训练模型在比训练时更长的长度工作。

**要点：**
- 注意力需要显式位置信号。
- RoPE 主宰现代 LLM。
- ALiBi 对更长序列外推良好。
- YaRN/NTK 缩放将 RoPE 拉伸到长上下文。

---

### 45. BERT vs GPT vs T5

**答案：** BERT（仅编码器）：掩码语言建模 + 下一句预测预训练，双向上下文；微调用于分类、NER、QA 跨度抽取；非生成设计。GPT（仅解码器）：带因果掩码的自回归下一 token 预测；原生生成；扩展到通用助手。T5（编解码器）：一切皆文本到文本；mask 跨度预训练；翻译、摘要、转为生成的分类等都灵活。今天多数基础模型为仅解码器（GPT-4、Claude、Llama、Mistral），因缩放效率和上下文学习的成功。仅编码器（BERT 家族）在嵌入/分类上仍以更低成本主宰。编解码器（T5、BART、FLAN-T5）在小规模翻译和指令遵循等保留 niche。

**要点：**
- BERT = 理解；GPT = 生成；T5 = 两者。
- 仅解码器赢得缩放竞赛。
- BERT 家族仍主宰廉价分类/嵌入。
- T5 在文本到文本任务微调中流行。

---

### 46. Vision Transformer（ViT）

**答案：** 把图像切成固定大小 patch（如 16x16），线性嵌入每个 patch + 加位置嵌入 + 前置 [CLS] token，然后跑标准 Transformer 编码器。[CLS] 嵌入（或池化的 patch 嵌入）驱动分类。在足够预训练（300M+ 图像如 JFT）下 ViT 超过 CNN；仅小数据下表现欠佳，因缺 CNN 归纳偏置（局部性、平移等变）。变体：Swin 用偏移窗口做层次化特征并近线性缩放；DeiT 用蒸馏做数据高效训练；DINOv2 大规模自监督。ViT 与后继者现主宰视觉基准，并支撑多模态模型（CLIP、GPT-4V、Gemini）。

**要点：**
- patch 当 token + 标准 Transformer。
- 需规模超过 CNN；仅小数据上弱。
- Swin/层次化变体提升效率。
- 现代多模态视觉编码器的基础。

---

### 47. GAN

**答案：** 生成器 G 把噪声 z 映射到数据；判别器 D 分真假。对抗训练：D 最大化正确判别，G 通过 D 的梯度最小化 D 的准确率。理论均衡为数据分布。实际中：mode collapse（G 忽略部分分布）、训练不稳定、无原则似然、对超参脆弱。主要变体：DCGAN（CNN 主干）、WGAN/WGAN-GP（Earth-Mover 距离，更稳）、StyleGAN（基于风格的生成器，人脸合成 SOTA）、条件 GAN（类或文本引导）、CycleGAN（无配对域翻译）。因稳定性和更好覆盖，2022 起在图像合成中多被扩散取代，但 GAN 推理仍快（单次前向）。

**要点：**
- 对抗 min-max 博弈。
- Mode collapse 与不稳定是经典失败。
- StyleGAN 家族长期保持人脸 SOTA。
- 2022 起扩散基本取代 GAN。

---

### 48. VAE

**答案：** 变分自编码器：编码器产生后验 q(z|x) 的参数（通常高斯），解码器从通过重参数技巧采样的潜变量 z 重建 x（z = mu + sigma * epsilon）。训练最大化 ELBO = 重建项 + q(z|x) 与先验 p(z)（通常单位高斯）的 KL 散度。KL 把潜变量推向先验，给出结构化生成潜空间；重建保保真。比 GAN 样本更模糊但稳定且给出原则性潜变量。变体：beta-VAE 做解耦，VQ-VAE 用离散潜变量（用于 DALL-E 1 和潜空间扩散模型如 Stable Diffusion 的自编码器）。Stable Diffusion 中的 VAE 编码器是其可扩展的主要原因——扩散在潜空间而非像素空间。

**要点：**
- ELBO = 重建 + KL。
- 重参数让采样可微。
- 比 GAN 模糊但更稳定。
- VQ-VAE 支撑潜空间扩散。

---

### 49. 扩散模型

**答案：** 前向过程：在数据上逐步加 T 步高斯噪声至纯噪声。反向过程：训练神经网络（通常 U-Net 或 DiT）预测每步的噪声，然后迭代去噪。用简单噪声预测 MSE 损失训练（Ho 等，DDPM）。采样：从噪声起，跑 T 步（或更少，通过 DDIM、DPM-Solver 等）反向过程。潜空间扩散（Stable Diffusion）在 VAE 潜空间中扩散以提速约 10 倍。通过 classifier-free guidance 做条件生成：联合训练有条件与无条件，采样时按 guidance 权重 w 缩放组合。现为图像/视频/音频合成的主导范式（DALL-E 3、SD3、Midjourney、Sora 风视频模型）。

**要点：**
- 学去噪；迭代反向过程。
- 潜空间扩散让高分辨率实用。
- Classifier-free guidance 做条件化。
- 替代 GAN 成为 SOTA 生成范式。

---

### 50. 扩散 vs GAN vs 自回归图像模型

**答案：** GAN：单次前向，推理快，图像锐利，但 mode collapse 且训练不稳。扩散：迭代去噪（多次前向），推理慢但训练易、模式覆盖广、通过 guidance 可控——保真和多样性的当前 SOTA。自回归图像模型（DALL-E 1、Parti、MAR）：像 VQ-VAE 码本上的语言模型顺序预测 token；灵活、通过 Transformer 易扩展，但成本二次且单向。一致性模型、flow matching 和 rectified flow 旨在结合扩散质量与 GAN 式单步生成。生产中：SD/SDXL/SD3 用于廉价开源图像，DALL-E 3/Midjourney 用于闭源质量，自回归用于统一多模态基础模型。

**要点：**
- GAN：快、不稳。
- 扩散：慢、稳、当前 SOTA。
- AR：像 LLM 那样扩展，易多模态统一。
- 单步扩散（一致性/rectified flow）缩小推理差距。

---

### 51. 自监督学习

**答案：** 用监督信号来自数据本身的前置任务从无标签数据训练表征。例如：掩码语言建模（BERT 预测隐藏 token）、下一 token 预测（GPT）、掩码图像建模（MAE 重建掩码 patch）、对比学习（SimCLR/CLIP 把增强拉近、其他推开）、bootstrap 方法（BYOL/DINO 用无负样本的目标网络）。SSL 通过利用海量无标签语料解锁现代基础模型。所得表征通过微调或线性探测转移到下游任务，常用少量标注数据。自监督预训练 + 监督微调是 NLP 和视觉的主导范式。小规模下前置任务质量比模型规模更重要。

**要点：**
- 从数据自身发明监督。
- 驱动 BERT、GPT、CLIP、DINO、MAE。
- 预训练 + 微调是现代工作流。
- 前置任务设计重要。

---

### 52. 迁移学习与微调

**答案：** 复用在大型通用数据集上预训练的模型用于较小相关任务。策略：特征抽取（冻主干、训新头）、全微调（用小 LR 更新所有参数）、逐层微调（自顶向下解冻）。NLP 中在分类、NER、QA 上微调 BERT/GPT。视觉中微调 ImageNet/JFT/CLIP 主干。关键技巧：预训练层用较低 LR（小 10-100 倍）、warmup、按层判别 LR、小数据冻结早期层、用正则化（dropout、权重衰减）避免灾难性遗忘。LLM 中参数高效方法（LoRA、adapter）以接近全微调质量加微小内存替代全微调。迁移学习是几乎所有现代 ML 应用的默认。

**要点：**
- 预训练主干 + 任务头 = 标准配方。
- 预训练权重用较低 LR。
- LoRA/PEFT 替代 LLM 的全微调。
- 通过谨慎 LR/冻结缓解灾难性遗忘。

---

### 53. LoRA 与参数高效微调（PEFT）

**答案：** 大模型全微调内存难承受。LoRA（Low-Rank Adaptation）冻结预训练权重并注入可训练低秩矩阵 A（d x r）和 B（r x d），使有效更新为 W + BA，其中 r << d（如 r=8-64）。仅训练约 0.1-1% 参数，许多任务质量近全微调。QLoRA 把基模量化到 4-bit 同时训练全精度 LoRA 适配器——单块 48GB GPU 上微调 70B 模型。其他 PEFT：adapter（每层小瓶颈模块）、prefix tuning（可学前缀 token）、prompt tuning（可学软提示）。LoRA 是 LLM 主导 PEFT 方法，因其简单、适配器可合并和生态支持。

**要点：**
- 训练低秩 delta 而非全权重。
- QLoRA：4-bit 基 + LoRA = 消费级 GPU 微调。
- 适配器可合并至基模实现零成本推理。
- 2023 起 LLM 定制的默认。

---

### 54. 知识蒸馏

**答案：** 训练小型"学生"模型匹配大型"教师"模型行为。损失结合硬标签交叉熵与教师输出分布上的软标签 KL 散度（常对 softmax 应用温度 T 软化概率以揭示暗知识）。变体：隐状态匹配、注意力匹配、生成的序列级蒸馏。用于压缩 BERT（DistilBERT、TinyBERT、MiniLM）、从前沿模型生产 LLM（Alpaca、Vicuna 蒸馏 GPT-4 输出）、创建端上模型。与量化和剪枝搭配良好。学生在相同数据上常比从头训练更好，因教师标签分布更平滑。关键：数据多样性对蒸馏质量比数量更重要。

**要点：**
- 匹配教师软输出，不仅硬标签。
- 温度软化概率以获更丰监督。
- DistilBERT、Vicuna 是经典示例。
- 与量化叠加获最大压缩。

---

### 55. 量化：INT8、INT4、GPTQ、AWQ

**答案：** 用更少位表示权重/激活以削减内存并加速推理。带校准的 INT8 训练后量化对多数模型近无损；INT4 需更谨慎。方法：GPTQ（逐层二阶优化以最小化重建误差）、AWQ（激活感知——保护显著权重通道）、bitsandbytes NF4（用于 QLoRA，对正态分布最优）、GGUF/llama.cpp 格式用于 CPU/GPU 推理。量化感知训练（QAT）将量化集成进训练以获最佳质量但昂贵。权衡：更小/更快但有些准确率损失，特别小模型在 INT4 时。现代开源 LLM 常在 INT4-8 服务；前沿任务（推理、代码）保持更高精度。结合 KV 缓存量化做长上下文节省。

**要点：**
- INT8 几乎免费；INT4 需谨慎。
- GPTQ 与 AWQ 是顶级训练后方法。
- QLoRA 用 NF4 基 + FP16 LoRA。
- 长上下文也量化 KV 缓存。

---

### 56. 分词：BPE、WordPiece、SentencePiece

**答案：** 将文本切为模型输入单元。Byte Pair Encoding（BPE）从字符起，迭代合并最频繁的相邻对；GPT 家族、RoBERTa 使用。WordPiece（BERT）类似但按最大化训练语料似然合并。SentencePiece（T5、LLaMA、Mistral）直接操作原始文本（含空格），无需预分词处理任何语言，支持 BPE 或 unigram LM。字节级 BPE（GPT-2 起）鲁棒处理任意字节，包括 emoji 和非拉丁脚本。权衡：词表大小（典型 32k-256k）影响序列长度和嵌入表大小。分词边界（数字、代码、非英语）驱动许多 LLM bug；现代模型加按位拆分数字或更大多语词表。

**要点：**
- BPE：贪心基于频率的合并。
- SentencePiece：语言无关、原始文本。
- 字节级 BPE 处理任意输入。
- 词表大小权衡序列长度与嵌入大小。

---

### 57. 词嵌入：word2vec、GloVe、fastText

**答案：** 把词映射到捕获语义相似的稠密向量。word2vec（Mikolov）通过 skip-gram（从词预测上下文）或 CBOW（从上下文预测词）加负采样训练——高效并产生类比（king - man + woman ≈ queen）有效的向量。GloVe 分解词共现矩阵以捕获全局统计。fastText 用子词（字符 n-gram）嵌入扩展 word2vec，处理 OOV 与形态丰富语言。静态嵌入（不论上下文每词一个向量）2018 前是 SOTA，之后被上下文嵌入（ELMo、BERT）取代。仍作为廉价特征和下游模型初始化有用；余弦相似度给出直观语义相关性。

**要点：**
- 每词一个向量，与上下文无关。
- word2vec 类比展示向量算术语义。
- fastText 处理子词与 OOV。
- 2018 后被上下文嵌入取代。

---

### 58. 上下文嵌入与句嵌入

**答案：** 上下文嵌入（ELMo、BERT、GPT）按句中上下文为词产生不同向量，捕获多义性（金融与河流中的 "bank"）。从预训练 Transformer 隐状态抽取。句级任务：池化 token 嵌入（mean、CLS、max），或用句相似性调优的模型。Sentence-BERT（SBERT）用 siamese/triplet 损失微调 BERT 产生句向量，余弦相似度匹配语义相似——远优于原始 BERT CLS。现代嵌入模型（OpenAI ada/text-embedding-3、BGE、E5、Nomic、Cohere）在大网页对上对比微调以产生高质量多语嵌入。语义搜索、RAG、聚类、去重、推荐的基础。

**要点：**
- 每 token 上下文相关向量。
- SBERT 风模型做句相似性。
- 现代嵌入模型在网页对上对比训练。
- RAG 与语义搜索的主干。

---

### 59. LLM 预训练目标

**答案：** 因果语言建模（CLM，下一 token 预测）：GPT 风解码器标配；简单、扩展优秀、原生用于生成。掩码语言建模（MLM）：从双向上下文预测掩码 token；BERT 家族使用，理解/分类好。Span corruption（T5）：掩码连续跨度，作为目标序列预测；结合双向编码与生成。Prefix LM：前缀双向，之后因果（UL2、GLM）。混合目标（UL2 的 MoD）在多目标上训练。经验上 CLM 在通用 LLM 上扩展最佳；MLM 在固定词表理解任务上小规模出色。预训练之外，模型用指令调优和 RLHF/DPO 进行后训练。

**要点：**
- CLM：通用 LLM 扩展。
- MLM：理解/分类最佳。
- Span corruption：T5 的中间道路。
- 混合目标改善 few-shot 迁移。

---

### 60. 指令调优

**答案：** 在多任务（FLAN、T0、Alpaca、ShareGPT、Open-Orca 等数据）的（指令、响应）对上微调预训练 LLM。教模型以 zero/few-shot 的方式跟随用户风格的自然语言指令，而非仅依赖原始预训练中的提示模式。本质上不增加知识；只是重塑交互风格并解锁基模的潜在能力。通常是 RLHF/DPO 后训练流水线的第一步。数据质量比数量更重要——LIMA 显示 1000 个精挑细选的样本就足以训练出强助手。陷阱：过拟合指令格式、失去多样性、在基模能处理的基准上退步。需与安全调优配套以抑制不需要的行为。

**要点：**
- 教格式，不是新事实。
- 数据质量 > 数量（LIMA）。
- 后训练管道第 1 步。
- 注意能力退步。

---

### 61. RLHF：来自人类反馈的强化学习

**答案：** 三阶段后训练流水线。阶段 1（SFT）：在精挑示范上做指令调优。阶段 2（奖励模型）：人类对同一提示的多个响应排序；训练模型预测偏好输出（典型用 Bradley-Terry 对数似然）。阶段 3（PPO）：用强化学习微调 LLM，最大化奖励模型分数，同时对 SFT 模型施加 KL 惩罚以防奖励黑客和分布坍塌。用于对齐 GPT-3.5/4、Claude、Gemini。优势：能处理示范难以具体说明的主观品质（有用性、无害性、风格）。弱点：流水线复杂、奖励黑客、mode collapse、过度奉承。DPO 及其后继者旨在通过移除显式 RL 步骤来简化。

**要点：**
- SFT → 奖励模型 → PPO。
- KL 惩罚控制分布漂移。
- 对齐示范之外的主观品质。
- 复杂；DPO 是现代更简单替代。

---

### 62. DPO 与偏好优化变体

**答案：** Direct Preference Optimization（DPO）把 RLHF 重新表述为分类问题：给定偏好对（选中、被拒），直接优化 LLM 对选中赋予比被拒更高似然，并隐式对参考模型 KL 惩罚。无奖励模型、无 PPO、无 rollout——仅在闭式损失上梯度下降。训练快得多、更简单、更稳，常与 PPO 质量相当。变体：IPO 修复 DPO 的偏好过拟合，KTO 用无对偶二元反馈，ORPO 一步结合 SFT+偏好，SimPO 去除参考模型。2024-2025 行业趋势是多数后训练用 DPO 家族，PPO 留给在线或过程监督设置。在消费级硬件加 LoRA 即可运行。

**要点：**
- 闭式偏好损失；无 RL。
- 比 PPO 更快、更简单、相当。
- IPO/KTO/ORPO/SimPO 是实用改进。
- 现为开源 LLM 对齐主导。

---

### 63. 上下文学习与 few-shot 提示

**答案：** LLM 可在推理时通过在提示中给出几个输入-输出示例（few-shot）或仅描述（zero-shot）执行新任务——无梯度更新。能力源自大规模预训练，并随模型规模和示例质量/多样性提升。示例的顺序、格式、标签分布有影响（近因偏置、多数标签偏置）。底层机制有争议，但部分涉及在上下文中复制模式的"induction heads"。ICL 推理时基本免费但占上下文 token，对高风险任务比微调不可靠。考虑 RAG 或微调前的强基线；提示工程的基础。

**要点：**
- 无参数更新——仅提示中示例。
- 对示例顺序、格式、标签平衡敏感。
- 尝试便宜；比微调不可靠。
- 常是强基线。

---

### 64. 提示工程技巧

**答案：** 结构化输入以引出更好 LLM 输出。核心技巧：清晰角色/任务陈述、显式格式/结构（JSON schema、XML 标签）、few-shot 示例、分解（把复杂分步）、自洽（采样多个、投票）、检索 grounding（RAG）、工具使用、结构化输出（函数调用、JSON 模式）。模式：思维链（"逐步思考"）、思维树、ReAct（推理 + 行动）、程序辅助 LM（用代码执行）。反模式：模糊指令、矛盾示例、引导性问题。现代 LLM 鲁棒到提示脆弱性下降，但高风险应用中系统化提示模板、评估套件和版本控制与模型选择同等重要。把提示当代码看。

**要点：**
- 对角色、格式、约束明确。
- CoT、自洽、ReAct 是首选模式。
- few-shot 示例塑造格式与行为。
- 提示值得版本控制与评估套件。

---

### 65. 思维链（CoT）推理

**答案：** 提示 LLM 在最终答案前生成中间推理步骤，如附加 "Let's think step by step."。最初是 few-shot 提示技巧（Wei 等，2022）；现已通过训练植入现代模型。改善数学、逻辑、多步推理表现，特别在更大模型中——在规模上涌现。自洽：采样多条 CoT 路径并多数投票最终答案——常大幅提升。Tree-of-Thought 推广为推理分支搜索。长 CoT / "推理模型"（OpenAI o1/o3、DeepSeek-R1、带 thinking 的 Claude）通过可验证奖励的 RL 训练模型生成长思维链，大幅改善数学/代码/STEM 但代价延迟和 token。2024-2026 向"思维"模型的转变使 CoT 成为一等能力。

**要点：**
- 中间步骤改善推理。
- 自洽 = 采样多个、投票。
- 推理模型（o1、R1）训练长 CoT。
- 代价延迟与 token；简单任务不一定值。

---

### 66. 上下文窗口与 KV 缓存

**答案：** 上下文窗口是 Transformer 可关注的最大序列长度——GPT-2 时代 2k，现前沿模型（Claude、Gemini）200k-2M。生成时 KV 缓存存储之前计算的 Key 和 Value 张量，每个新 token 只关注缓存 K/V 而非重新计算——把每 token 的 O(n^2) 变 O(n)。KV 缓存随上下文长度线性增长，长上下文时主导内存（如 LLaMA-70B 在 32k 约几十 GB）。优化：GQA/MQA 减 KV 头，paged attention（vLLM）像虚拟内存管理 KV，量化 KV 缓存（INT8/FP8），提示缓存（带共享前缀的调用复用缓存），滑动窗口注意力丢老 KV。长上下文推理经济学由 KV 缓存管理主导。

**要点：**
- KV 缓存让每 token 生成 O(n) 而非 O(n^2)。
- 长上下文时内存由 KV 主导。
- GQA、paged attention、量化减 KV 成本。
- 提示缓存跨请求复用前缀 KV。

---

### 67. 注意力扩展：FlashAttention、稀疏、线性

**答案：** 朴素注意力因 n x n 注意矩阵在计算与内存上 O(n^2)。FlashAttention（Dao 等）通过 tiling + 重计算重组计算避免物化完整矩阵，获 2-4 倍加速与重大内存节省——现为 PyTorch/CUDA 内核标配。稀疏注意力（BigBird、Longformer）限制注意力到局部窗口 + 全局 token，以表达力代价获亚二次复杂度。线性注意力（Performer、Linformer）用核近似 softmax 做 O(n) 成本；质量常落后。状态空间模型（Mamba、Mamba-2）用选择性 SSM 完全替代注意力，在语言上获线性时间训练与有竞争力质量。混合注意力 + SSM（Jamba、Zamba）融合两者。该领域为很长上下文积极转向亚二次替代。

**要点：**
- FlashAttention：相同数学，IO 感知，2-4 倍快。
- 稀疏/线性注意力降复杂度但损质量。
- Mamba/SSM 是涌现的线性替代。
- 混合架构是实用妥协。

---

### 68. 专家混合（MoE）

**答案：** 用 N 个专家 FFN 加一个路由器替换每个 FFN，路由器为每个 token 选 top-k（典型 k=1 或 2）专家。总参数扩展到数千亿，而每个 token 计算仅用其中 k/N——如 Mixtral 8x7B 约 47B 参但每 token 只用约 13B。训练需要负载均衡损失以防路由器坍塌（所有 token 都涌向一个专家）并需要路由技巧（expert capacity、switch transformer）。优势：每 FLOP 质量更好、比稠密模型更易扩参。劣势：内存更多（所有专家都要加载）、高效服务更难、训练更复杂。用于 Mixtral、DeepSeek-V3（256 专家）、GPT-4（传闻）、Grok-1、DBRX。高效扩展前沿。

**要点：**
- 稀疏激活：参数巨大、计算适度。
- 每 token top-k 路由。
- 负载均衡防坍塌。
- 现代前沿：DeepSeek-V3、Mixtral、GPT-4 级。

---

### 69. MQA 与 GQA：高效 KV

**答案：** 标准多头注意力有 h 个 Q 头、h 个 K 头、h 个 V 头。Multi-Query Attention（MQA，Shazeer 2019）所有头共享一个 K 和一个 V——以小质量代价削减 KV 缓存与内存带宽约 h 倍。Grouped-Query Attention（GQA）是中间道路：h 个 Q 头、g 个 K/V 头（g = h / group_size）。LLaMA-2/3 用 g=8 配 32 或 64 个 Q 头，获多数 MQA 加速且几乎无质量损失。对 KV 缓存主导内存的长上下文推理关键。现为基本上所有现代开源 LLM 标配。从全 MHA 转 GQA 可通过轻微调事后完成。

**要点：**
- MQA：共享单个 K/V；削减缓存约 h 倍。
- GQA：分组 K/V 头；甜蜜点。
- LLaMA-2/3、Mistral、现代 LLM 标配。
- 长上下文推理经济关键。

---

### 70. 长上下文技巧

**答案：** 因 O(n^2) 注意力，长序列预训练昂贵。扩展策略：位置插值（线性缩放 RoPE 位置）、NTK-aware 缩放（频域插值）、YaRN（带注意力温度的更好 NTK）通过短暂微调把 4k/8k 训练模型扩到 32k-128k。滑动窗口注意力（Mistral）保留最近 K/V 加注意力 sink（StreamingLLM 保留作为偏置的初始 token）。推理时，提示缓存跨请求复用前缀 KV；分块预填充批处理长提示处理。架构替代：SSM（Mamba）、混合模型（Jamba）、检索以使所需上下文更短。尽管头条上下文长度 1M+ token，有效上下文（模型实际用远距信息处）常更短——"lost in the middle" 问题。

**要点：**
- RoPE 缩放（YaRN/NTK）扩展预训练模型。
- 滑动窗口 + 注意力 sink 做流式。
- "Lost in the middle" 是真实质量悬崖。
- RAG 常胜过把所有都塞进上下文。

---

### 71. LLM 中的幻觉

**答案：** LLM 有时产生自信、流利但事实错误或编造的文本（引用、统计、代码）。原因：预训练数据错误、优先合理性的自回归采样、缺失知识由插值填充、指令调优偏好总是回答。缓解：检索增强生成（用检索来源 grounding 答案）、引用/归属提示、结构化输出（受限解码）、思维链 + 验证、自洽、放弃训练（"我不知道"）、不确定信号（logprob、语义熵）和事后事实检查器（LLM-as-judge 或基于检索）。前沿模型幻觉更少但仍有，特别在 niche 主题、近期事件和对抗提示上。幻觉抗性是与推理和指令遵循并列的模型评估关键轴。

**要点：**
- 自信、合理但错误的输出。
- RAG 与引用是顶级缓解。
- 自洽与放弃有帮助。
- 把幻觉当一等指标评估。

---

### 72. 检索增强生成（RAG）

**答案：** 查询时从语料检索相关文档（通常通过嵌入向量搜索）并注入 LLM 提示作为上下文。让模型回答私有、最新或大型语料而无需重训。管道：文档分块 → 嵌入 → 存向量 DB → 查询时嵌入查询、检索 top-k → 可选重排 → 组装提示 → 生成（带引用）。与微调权衡：RAG 原生处理时效与归属，微调把模式/风格烘进模型。多数生产 "LLM 应用"用 RAG。失败模式：检索遗漏、分块边界切断上下文、不相关检索混淆模型、通过检索内容做提示注入。检索质量常比 LLM 选择更重要。

**要点：**
- 检索把生成 grounding 到外部源。
- 管道：分块 → 嵌入 → 检索 → 重排 → 生成。
- 事实/时效上比微调好。
- 检索质量常是瓶颈。

---

### 73. 向量数据库与 ANN 搜索

**答案：** 存储并索引高维嵌入做快速最近邻搜索。精确搜索 O(n*d)；近似（ANN）以小召回损失换数量级加速。算法：HNSW（层次可导航小世界图，低延迟、高召回）、IVF（带 k-means 分区的倒排文件，可扩展）、IVF-PQ（乘积量化压缩向量）、ScaNN（Google）、FAISS（Meta 库，全部上述）。向量 DB：Pinecone、Weaviate、Qdrant、Milvus、pgvector、OpenSearch、Elasticsearch dense vector、Chroma。按规模、召回需求、过滤需求（元数据）、混合搜索（向量 + 词汇）和运维模型（托管 vs 自托管）选择。小数据（<1M 向量）内存 FAISS 甚至蛮力即可。

**要点：**
- HNSW：低延迟、高召回默认。
- IVF-PQ 用于压缩大规模。
- 混合（向量 + BM25）常胜纯向量。
- pgvector 简单；Pinecone/Qdrant 用于规模。

---

### 74. RAG 的分块策略

**答案：** 把文档切成可检索块影响下游一切。策略：固定字符/token 块（简单）、递归切分（先段落、再句子、再字符）、语义分块（在嵌入相似边界切）、结构化（标题、章节、代码块）、文档特定（LaTeX、代码、表格）。典型块大小：200-1000 token，10-20% 重叠。权衡：小块给精确检索但失去上下文；大块带更多上下文但稀释相关性和浪费 token。加元数据（来源、章节、日期）做过滤和引用。父文档检索嵌入小块但返回更大周围上下文。Late chunking（Jina）嵌入整文再按块池化以保留全局上下文。分块选择可大幅波动 RAG 质量。

**要点：**
- 递归或语义分块胜过朴素切分。
- 重叠（10-20%）防上下文切断。
- 父文档检索弥合精确/上下文权衡。
- 元数据使过滤和引用成为可能。

---

### 75. RAG 中的重排

**答案：** 初始向量检索快但粗；重排器用更准（更慢）模型重打分 top-k 候选以改善精确度。Cross-encoder 重排器（如 BGE-reranker、Cohere Rerank）把（查询、文档）作为联合输入并输出相关分——比 bi-encoder 余弦相似度区分细微差异好得多。典型管道：通过向量 + BM25 混合检索 top-50/100、用 cross-encoder 重排、保留 top-5/10 给 LLM。加延迟（每候选数十 ms）但常是 RAG 中检索本身后最大质量提升。开源：BGE-reranker、mxbai-rerank、Jina reranker。托管：Cohere、Voyage。LLM-as-reranker（用小 LLM 打分）也可行但贵。

**要点：**
- Cross-encoder 比嵌入相似更准。
- 检索宽，重排窄。
- 常是最高 ROI 的 RAG 改进。
- 加延迟但大精确度收益。

---

### 76. 混合搜索：向量 + 词汇

**答案：** 纯向量搜索错失精确匹配情况（缩写、标识符、稀有词）；纯 BM25/词汇错失语义匹配。混合搜索通过分数融合结合：reciprocal rank fusion（RRF，简单无参，实践常最佳）、加权归一化分、或学习融合。多数生产 RAG 系统用混合，因典型查询混合精确词（产品名、错误码）和概念。实现：Elasticsearch/OpenSearch hybrid、Weaviate、Qdrant、Vespa、Milvus。ColBERT（late interaction）是中间地带，以向量效率做细粒度 token 级匹配。混合在生产中基本总优于仅向量，仅适度基础设施开销。

**要点：**
- 向量做语义，BM25 做精确 token。
- RRF 是简单强融合。
- 严肃 RAG 的生产默认。
- ColBERT：token 级 late interaction。

---

### 77. 智能体与工具使用

**答案：** LLM 智能体通过交错推理、工具调用（搜索、代码执行、API）和观察超越单次生成。模式：ReAct（推理 + 行动轨迹）、函数/工具调用（结构化 JSON 输出触发工具，结果反馈）、计划与执行（规划器制定计划，执行器跑步骤）、反思（自批评后修订）、多智能体（专门智能体协作）。现代 LLM（GPT-4、Claude、Gemini）暴露一等函数调用 API。框架：LangChain、LlamaIndex、Anthropic tool use、OpenAI assistants、AutoGen、CrewAI。生产挑战：延迟（许多往返）、成本（长轨迹）、可靠性（循环、错误）、评估（多步轨迹难评分）、安全（工具误用）。最适合下一步取决于前一结果的任务。

**要点：**
- 推理 → 调工具 → 观察 → 重复。
- 函数调用是标准接口。
- 延迟、成本、评估是实用挑战。
- 多步自适应 > 许多任务单次。

---

### 78. 函数调用与结构化输出

**答案：** LLM 训练在用户提供工具/函数规范时发出结构化输出（匹配 schema 的 JSON）。实现：提供函数名、描述、参数 JSON schema；模型决定是否/哪个调用、发出参数，应用执行并返回结果给模型用。现代提供商（OpenAI、Anthropic、Google）原生提供。为可靠：受限解码（通过 Outlines、Instructor、OpenAI strict mode 等库强制输出匹配语法/JSON schema）、schema 验证带重试、系统提示中示例。对智能体、带元数据过滤的 RAG、表单填写、从非结构化文本 ETL 关键。模式：tool-choice "auto" 让模型决定；"required" 强制调用；"none" 禁用。

**要点：**
- Schema 定义的工具；模型发 JSON 参数。
- 受限解码确保有效输出。
- 智能体与结构化工作流的基础。
- schema 错误时验证并重试。

---

### 79. LLM 评估：基准与 LLM-as-judge

**答案：** 两类。自动化基准：MMLU（多任务知识）、HumanEval/MBPP（代码）、GSM8K/MATH（数学）、HellaSwag（常识）、TruthfulQA（真实性）、MT-Bench（聊天）。前沿模型饱和；游戏化和污染担忧。应用特定评估：构建与你任务挂钩的自定义测试集（如 "RAG 答案是否匹配文档？"）。LLM-as-judge：用强 LLM 按 rubric 评分输出，比人评便宜得多，提示谨慎时相关性好——但有偏置（位置、冗长、自偏好）。最佳实践：小人工标注金集 + LLM-as-judge 做规模 + 抽查。每次模型/提示变更追踪退化。评估是 LLM 应用最被忽视的部分。

**要点：**
- 公共基准：有用但饱和/污染。
- 与你任务挂钩的自定义评估最重要。
- LLM-as-judge 廉价扩展评估。
- 把评估当版本化产品工件。

---

### 80. RAG 系统评估

**答案：** RAG 质量分解：检索（找到正确文档了吗？）、生成（正确使用它们了吗？）。检索指标：recall@k、MRR、NDCG、命中率（金文档在 top-k 吗）。生成指标：忠实度（无超出上下文的幻觉）、答案相关性、答案对参考的正确性。框架：RAGAS、TruLens、DeepEval。合成评估集生成：让 LLM 从你的文档创建 QA 对，再针对它们评估检索 + 生成。务必为高风险决策结合小型人工金集。常见发现：检索通常是瓶颈；分块和重排改进比换 LLM 移动指标更多。生产中通过用户反馈信号（拇指、跟进点击）监控。

**要点：**
- 分别评估检索 vs 生成。
- RAGAS/TruLens 自动化常见指标。
- 合成 + 人工金集。
- 检索通常是瓶颈。

---

### 81. 特征存储

**答案：** 训练和推理间存储、服务、共享 ML 特征的中心化服务。解决：训练/服务偏移（特征在训练 vs 生产中计算不同）、跨团队特征复用、时间感知特征的时间点正确性（训练标签必须用标签时存在的特征，非 "现在"）。架构：离线存储（仓库/湖、批特征）做训练，在线存储（键值，如 Redis、DynamoDB）做低延迟服务，两者由同一特征管道喂养。工具：Feast、Tecton、Hopsworks、Databricks Feature Store、SageMaker Feature Store、Vertex AI Feature Store。许多模型共享特征或时间点正确性重要时值得；一次性模型过度。

**要点：**
- 防训练/服务偏移。
- 离线（训练）+ 在线（服务）存储。
- 时间感知标签的时间点连接。
- 超过几个生产模型后值得。

### 82. 实验追踪与模型注册

**答案：** 实验追踪记录每次训练运行的代码、数据版本、超参、指标和工件，使实验可复现可比较。工具：MLflow、Weights & Biases、Neptune、Comet、Aim。模型注册存储带元数据、阶段（staging、production、archived）、血缘（产生它的运行）和审批状态的版本化模型。它们共同构成 MLOps 基础：每个生产模型可追溯到特定运行、数据集和代码 commit。与 CI/CD 集成，使生产晋升触发测试、评估、金丝雀部署。无追踪时 "哪个模型在生产，怎么训练的？" 无法回答——ML 组织痛苦的主因。

**要点：**
- 追踪代码 + 数据 + 参数 + 指标 + 工件。
- 注册晋升/批准的模型版本。
- 可复现 = 从生产到 commit 的血缘。
- MLflow/W&B 是常见默认。

---

### 83. ML 模型 A/B 测试

**答案：** 通过随机分流用户/流量比较候选模型与当前生产。定义主要成功指标（营收、留存、CTR）和护栏指标（延迟、错误率、公平性）。基于最小可检测效应（MDE）和预期基线方差选样本量做统计功效。运行全业务周期（不要在每周周期应用的周一结束），检查新颖效应。用顺序或贝叶斯测试安全偷看。注意：SUTVA 违反（网络效应）、跨段的辛普森悖论、跨多指标的多重比较膨胀。务必 A/B 后再宣布成功——离线指标常与线上不一致。Holdout cohort 测短 A/B 窗口外的长期效应。

**要点：**
- 线上 > 线下；信模型前先 A/B。
- 主要指标 + 护栏。
- 功效分析定样本量。
- 注意 SUTVA、新颖、多重比较。

---

### 84. 金丝雀部署与影子模式

**答案：** ML 模型推出的风险缓解模式。影子模式：把生产流量并行发到新模型与当前模型，记录预测，离线比较——不影响用户。捕获基础设施 bug 和粗分布偏移。金丝雀部署：路由小百分比（1%、5%、25%）真实流量到新模型，监控业务和护栏指标，健康则提升，不健康则快速回滚。与特性开关结合做细粒度控制。对 ML 特别重要，因为模型可能以微妙、分布依赖的方式失败而离线评估错过。构建关键指标退化时的自动回滚。与 A/B 配对做改进的统计证据。

**要点：**
- 影子：仅记录，不影响用户。
- 金丝雀：小活流量 + 自动回滚。
- 特性开关 + 指标 + 回滚自动化。
- 捕获离线评估错过的。

---

### 85. 模型监控与漂移检测

**答案：** 生产模型随输入偏移悄然退化。监控：数据漂移（输入特征分布 vs 训练）、概念漂移（X 与 y 关系变）、预测漂移（模型输出分布移）、性能漂移（准确率，当 ground truth 延迟可用时）。方法：PSI、KL 散度、Kolmogorov-Smirnov、ADWIN 流式、图像/文本嵌入漂移。按特征和按段追踪，不仅全局。设带迟滞警报避免噪声。延迟、错误率、吞吐量也需监控。工具：Evidently、Arize、WhyLabs、Fiddler、Datadog ML。无重训策略的漂移检测只是警报；与自动重训触发或人工审查配套。

**要点：**
- 数据、概念、预测、性能漂移不同。
- PSI/KL 测分布偏移。
- 按段监控捕获局部退化。
- 检测与重训或人工分诊配套。

---

### 86. 批 vs 实时模型服务

**答案：** 批：离线（夜间、每小时）计算预测并存数据库/缓存供请求时读取。更简单、更便宜、易扩展；时效需求小时到天时合适（流失分、推荐、线索打分）。实时/在线：模型每请求查询，同步算预测。新鲜输入需要（搜索排名、欺诈、广告、个性化）。架构：HTTP/gRPC 服务、自动扩展、p99 延迟预算（常 <100ms）、模型内存加载、批微批做吞吐。混合：批中预计算特征，上面跑轻量实时模型。按延迟要求、输入新鲜度和成本选。多数生产 ML 系统混合两者。

**要点：**
- 批：更便宜、更简单、时效小时 OK。
- 实时：每请求、亚秒延迟。
- 混合：批特征 + 实时打分。
- 延迟 SLO 驱动架构。

---

### 87. LLM 推理引擎：vLLM、TGI、TensorRT-LLM

**答案：** 高吞吐 LLM 推理的专用服务器。vLLM 引入 PagedAttention（KV 缓存像虚拟内存页处理），通过更好内存利用率和连续批处理（序列完成时插入/移除批，不等最慢）相对朴素服务 2-24x 吞吐。TGI（Hugging Face Text Generation Inference）是可比生产服务器。TensorRT-LLM（NVIDIA）用核融合、量化和图优化编译模型获最大 GPU 吞吐。SGLang 加 RadixAttention 做前缀共享。共同关键特性：连续批处理、paged KV、量化（INT8/FP8/INT4）、speculative decoding、前缀缓存、multi-LoRA 服务。选择视模型、硬件和运维契合。vLLM 是多数团队的开源默认。

**要点：**
- 连续批处理 + paged KV = 吞吐倍增。
- vLLM 主导开源选择。
- TensorRT-LLM 获最大 NVIDIA 性能。
- 前缀缓存对重复系统提示巨大。

---

### 88. 推测性解码

**答案：** 通过用小 "draft" 模型提前提议 K 个 token，然后让大模型单次前向并行验证（或拒绝/纠正）draft 来加速 LLM 生成。接受的 token 等于"免费"；拒绝的成本也不超过朴素生成。获得 2-4x 加速且质量不变（输出分布与从目标模型贪心/温度采样相同）。变体：基于树（Medusa，每步多提议）、自我推测（用模型自身早期层作 draft）、EAGLE（带隐状态输入的更好 draft 头）。对低延迟交互式 LLM 服务至关重要。大 batch size 时收益递减（受延迟约束更弱）。现已成为 vLLM、TensorRT-LLM 和推理 API 的标配。

**要点：**
- Draft 小、验证大；相同输出分布。
- 典型 2-4x 加速，无质量损失。
- Medusa/EAGLE 是更强变体。
- 小批最佳（受延迟约束）。

---

### 89. 连续批处理与 paged attention

**答案：** 静态批处理把所有序列填充至最大长度并一起处理——序列在不同时间完成时浪费计算。连续批处理（Orca、vLLM）一旦其他完成 token 就把新请求插入批，对变长生成显著改善 GPU 利用率。Paged attention 把 KV 缓存当固定大小页（像 OS 虚拟内存）处理，消除碎片并使带共同前缀（系统提示、few-shot 示例）的请求间高效内存共享。它们共同使生产 LLM 服务 10x+ 吞吐相对朴素方法。以内存管理复杂换吞吐。vLLM、SGLang、TGI、TensorRT-LLM 标配。

**要点：**
- 连续批处理：短序列不空闲 GPU。
- Paged KV：无碎片、前缀共享。
- 相对朴素服务 10x+ 吞吐。
- 现代 LLM 服务器标配。

### 90. 生产模型蒸馏

**答案：** 把大教师模型压成更小生产学生。方法：标准 KD（用温度匹配 logit）、序列级蒸馏（在教师生成输出上训练）、任务特定（仅在你任务的教师预测上训学生——常远好于通用蒸馏）。LLM：用前沿模型（GPT-4、Claude）的合成数据微调小开源模型（Mistral 7B、Qwen）。注意许可（一些提供商禁止用输出训竞争模型）。与量化和剪枝结合获最大压缩。实用配方：用 LoRA 把你领域的 GPT-4 输出蒸馏进 Llama-3-8B，单 GPU 部署，获前沿质量 90%+ 而成本 1%。2024-2026 主导生产化路径。

**要点：**
- 任务特定蒸馏胜过通用。
- 前沿合成数据 → 小学生。
- 注意提供商关于输出的 TOS。
- 与量化叠加获最大压缩。

### 91. 边缘与端上 ML

**答案：** 在手机、浏览器、嵌入式设备运行模型。约束：内存、计算、电池有限，无 GPU。栈：TensorFlow Lite、ONNX Runtime、Core ML（iOS）、MediaPipe、llama.cpp、MLC-LLM、Apple 的 Foundation Models、Qualcomm AI Engine。技术：量化（INT8/INT4 默认，有时二进制）、剪枝（硬件结构化）、蒸馏、移动优化架构（MobileNet、EfficientNet、MobileBERT）、目标硬件的神经架构搜索。端上 LLM：1-8B 参数模型 INT4 装入手机 RAM（Phi-3-mini、Gemma 2B、Llama 3.2 1/3B）。优势：隐私（数据不离设备）、离线、低延迟、零每请求成本。挑战：模型大小、热降频、OS 碎片。2025-2026 趋势是带智能路由的混合云-边缘。

**要点：**
- 紧张的内存/计算/电池预算。
- 量化 + 小架构必需。
- 1-8B LLM 在现代手机可行。
- 隐私 + 离线是关键优势。

---

### 92. GPU 效率与训练成本

**答案：** 训练成本 = (模型 FLOPs * token) / (GPU FLOPs * 利用率 * 时间)。实用优化：混合精度（FP16/BF16/FP8）2-4x 加速，梯度检查点以计算换内存，ZeRO/FSDP 跨 GPU 分片优化器/梯度/参数，张量/流水线/数据并行做多节点，FlashAttention 实现内存高效注意力，梯度累积达成更大的有效批。监控：MFU（Model FLOPs Utilization），Transformer 训练典型 40-55%。前沿训练单次成本可达几千万到几亿美元；效率收益会复利。推理：量化、批处理、paged attention、推测性解码、multi-LoRA 服务。Chinchilla 缩放律指导模型大小与训练 token 间的计算最优分配。

**要点：**
- BF16/FP8 + FlashAttention + FSDP = 现代栈。
- MFU 是头条效率指标。
- Chinchilla：平衡参数 vs token。
- 推理成本由 KV/批处理/量化主导。

---

### 93. 缩放律与 Chinchilla

**答案：** Kaplan 等（2020）展示模型性能在计算、数据、参数上服从幂律缩放。Chinchilla（Hoffmann 等，2022）纠正了最优计算分配：给定计算预算 C，参数 N 和 token D 应大致同比例缩放（D ≈ 20N），使先前的 "巨模型、适度数据" 运行（GPT-3、PaLM）次优。LLaMA、Mistral、现代模型在更多 token 上训练更小模型（LLaMA-3 8B 在 15T token），相对 Chinchilla 过训但更好推理成本。缩放律跨数量级成立直到数据质量主导。Chinchilla 后洞见：推理重场景中，训更小模型更久比更大模型更短好。缩放律指导前沿训练预算分配。

**要点：**
- 计算、数据、参数上的幂律。
- Chinchilla：计算最优每参约 20 token。
- 过训更小模型削减推理成本。
- 数据质量最终比规模更重要。

---

### 94. 强化学习基础

**答案：** 智能体通过行动与环境交互，接收观察和奖励；学习最大化期望折扣回报的策略 pi(a|s)。关键概念：状态值 V(s)、动作值 Q(s,a)、折扣因子 gamma、探索 vs 利用（epsilon-greedy、UCB、熵奖励）、on-policy（PPO、A2C）vs off-policy（DQN、SAC）、无模型 vs 基于模型。算法：Q 学习 + 神经网络 = DQN，策略梯度 = REINFORCE，actor-critic = A2C/A3C/PPO/SAC。挑战：样本低效、奖励设计（稀疏奖励、奖励黑客）、信用分配、不稳定。RL 在游戏（Atari、围棋、星际争霸）、带模拟的机器人和 LLM 后训练（RLHF/PPO、推理模型的 RLVR）中最成功。多数 "实用" 生产 RL 是 bandit，非全 RL。

**要点：**
- 通过策略最大化期望折扣奖励。
- PPO 是首选通用算法。
- 样本低效是慢性问题。
- bandit 常比全 RL 更实用。

---

### 95. 多模态模型：CLIP、GPT-4V、Gemini

**答案：** 多模态模型联合处理图像、文本、音频、视频。CLIP 在巨大网络抓取的图像-文本对上对比训练（匹配图像嵌入与标题嵌入），实现 zero-shot 分类（"哪个类标签在嵌入空间最近？"）并驱动检索/标题。GPT-4V、Claude 3+、Gemini 在统一上下文中原生接受图像和文本，回答视觉问题、OCR、读图表、从截屏写代码。架构上多通过 projector（Q-Former、MLP）把视觉编码器（ViT、CNN）融入 LLM 再传 Transformer。原生多模态训练（Gemini、GPT-4o）在文本/图像/音频交错 token 上联合训练。2025-2026 前沿扩展到视频和实时语音，音频-语言模型（Whisper、GPT-4o voice）成一等。

**要点：**
- CLIP：对比图像-文本，检索基础。
- VLM：视觉编码器 + projector + LLM。
- 原生多模态（Gemini、GPT-4o）联合训练。
- 2025-2026 前沿：视频、实时语音。

### 96. AI 对齐

**答案：** 确保 AI 系统追求与人类价值和意图对齐的目标。子问题：外部对齐（指定我们想要的——奖励建模、constitutional AI、辩论）、内部对齐（模型实际优化我们指定的，非代理——mesa 优化、欺骗性对齐）、可扩展监督（人类不能直接评估超人输出——用 AI 助手、辩论、递归奖励建模、weak-to-strong 泛化）。今日实用技术：RLHF、DPO、constitutional AI（模型对照原则自我批评）、red-teaming、评估。前沿实验室（Anthropic、OpenAI、DeepMind、Meta）都有对齐团队。先进 AI 的开放问题：欺骗性对齐、目标保留、可纠正性、可解释性。随模型变智能体和有能力自主采取后果行动，对齐更重要。

**要点：**
- 外部（规范）+ 内部（优化）对齐。
- RLHF/DPO/constitutional AI 是实用工具。
- 可扩展监督是开放研究领域。
- 风险随自主性和能力上升。

### 97. 提示注入与越狱

**答案：** 操纵 LLM 绕过安全准则或遵循攻击者指令的对抗输入。直接越狱：聪明提示（"忽略前面指令"、DAN、角色扮演）绕过拒绝。间接提示注入：检索文档、网页、邮件、图像中隐藏的恶意指令，智能体把它们当系统指令。在带工具访问的智能体系统中日益危险（外泄数据、采取未授权行动）。缓解：系统提示层次（指令遵循层次，训练模型不信任不可信内容）、输入/输出过滤器、内容来源标记、敏感行动带人在环的沙箱工具使用、约束行为的结构化输出、dual-LLM 模式（特权 + 沙箱）。无技术完全可靠；深度防御是唯一实用答案。

**要点：**
- 直接（用户）vs 间接（检索内容）注入。
- 带工具的智能体大幅提升风险。
- 指令层次 + 过滤 + 沙箱。
- 默认把检索内容当不可信。

### 98. LLM 基准：MMLU、HellaSwag、HumanEval、MATH

**答案：** 追踪 LLM 能力的公共基准。MMLU：57 个科目的多任务知识；前沿模型 90%+（饱和）。HellaSwag：常识句子补全；饱和。HumanEval/MBPP：Python 函数生成；SWE-Bench：真实 GitHub issue，难得多。GSM8K：小学数学应用题；饱和。MATH/AIME：竞赛数学，推理模型（o1、R1）闪光。ARC-AGI：抽象推理，故意对 LLM 难。MT-Bench、Arena Hard、Chatbot Arena：聊天质量。GPQA：研究生科学。开放：训练数据大量污染、向基准调优过拟合。实时排行榜（LMSys Arena、LiveBench、SimpleBench）通过新提示减污染。信你应用特定评估胜任何公共基准。

**要点：**
- MMLU、HellaSwag、GSM8K 大体饱和。
- SWE-Bench、MATH、GPQA 仍区分前沿模型。
- 污染 + 基准过拟合真实。
- 你的自定义评估 > 公共基准。

### 99. 负责任 AI 与偏见缓解

**答案：** ML 模型继承并放大训练数据偏见（种族、性别、年龄、地理）。例：深肤色人脸识别错误率高得多；简历筛选偏爱男性名字；一类人群训练的医学模型对其他失败。公平准则：人口平等（跨组等结果）、机会平等（等 TPR）、equalized odds（等 TPR 和 FPR）——常互斥。生命周期缓解：代表性数据、偏见审计、公平感知算法（重加权、对抗去偏、后处理）、生产中按组监控指标、记录限制的模型卡/数据表。监管：欧盟 AI 法、美国行政令、行业特定规则。负责任 AI 也覆盖隐私（DP、联邦学习）、安全、透明和人类监督。不能事后改造——从设计植入。

**要点：**
- 偏见通过 ML 管道复合。
- 公平指标常互不兼容。
- 按组监控捕获局部伤害。
- 监管（欧盟 AI 法）正变约束性。

### 100. LLM 端到端生产化

**答案：** 生产 LLM 应用结合：模型选择（前沿 API vs 开源自托管 vs 蒸馏小模型）、提示工程 + 版本控制、带向量 DB + 混合搜索 + 重排的 RAG、结构化输出/函数调用、需要自适应控制流处的智能体/工具使用、评估（自动化基准 + LLM-as-judge + 人工金集 + 生产反馈）、护栏（输入/输出过滤、PII 脱敏、提示注入防御）、可观测性（延迟、成本、token 用量、错误率、幻觉信号）、服务（vLLM/TGI、自动扩展、缓存、批处理、量化）、持续改进（A/B 测试、微调循环、提示迭代）、成本管理（模型路由——简单用便宜，难用昂贵）。模型日益是廉价商品；数据、评估、检索和运维才是生产质量赢得或失败之处。

**要点：**
- 模型是商品；数据 + 评估 + 运维差异化。
- RAG + 结构化输出 + 护栏是基本。
- 可观测性和评估循环驱动改进。
- 按难度路由：简单用小模型，难用前沿。
